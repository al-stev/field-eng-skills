# Design Critique: Customer Tracker Dashboard (2026-03-11)

Generated via /impeccable-critique on customers/g-research/trackers/2026-03-11-dashboard.html

## Anti-Patterns Verdict: PASS

Avoids all major AI slop tells: no Chart.js, no KPI cards, no gradient text, no glassmorphism, no purple-to-blue gradients, no neon accents. Distinctive type stack (Instrument Serif + Outfit + JetBrains Mono). Tinted neutrals, gold accent, noise texture.

## What's Working

1. **Callout severity system**: Left-border color coding (red/orange/amber/blue) with stacked layout feels like a log severity system an engineer would trust.
2. **Activity-based health classification**: Computed buckets (Needs Triage / Active / Stale / Resolved) based on eng comment activity, with FE-UPDATE exclusion.
3. **Typography discipline**: Three fonts for exactly the right roles. Mono micro-labels create consistent data-presentation layer.

## Priority Fixes (deferred)

### 1. Callout "View tickets" invisible until hover
- Primary interactive affordance relies on hover-only label (opacity: 0 -> 1)
- Users won't know callouts are clickable without accidentally hovering
- On touch devices, the label never appears
- **Fix**: Make always visible at opacity 0.5, brighten on hover. Or add persistent right-arrow.

### 2. No resolved issues toggle visible
- Design system specifies Show/Hide Resolved toggle but it's not rendered
- Resolved excluded via status filter default, but no obvious toggle
- **Fix**: Add dedicated "Show Resolved" pill in filter bar

### 3. 57% Uncategorized dominates theme chart
- 48/84 issues are Uncategorized, overwhelming meaningful themes
- **Fix**: Put Uncategorized last (not sorted by count), dim visually, add explanatory note

### 4. Velocity chart zero-month handling
- Months with 0 opened/resolved may render empty or be skipped
- **Fix**: Always render all 6 months with empty bars for zero values

## Minor Observations

- Triage bucket (orange) and "no eng activity" callout (medium/orange) create visual ambiguity
- 640px responsive breakpoint hides columns 6-7 -- verify these are least important
- Recently Opened uses 30 days in JS but .continue-here.md says 14 days -- mismatch
- expandCollapseAll button (10px font, 3px padding) is very small

## Questions to Consider

- Should health bucket clicks compete with callout clicks as filter paradigms?
- Would a single summary sentence be more scannable than bucket numbers for SE presentations?
- Is the 6-month velocity chart pulling its weight vs a simpler quarterly ratio?
