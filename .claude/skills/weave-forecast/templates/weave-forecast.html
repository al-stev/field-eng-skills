Scenario summary
Cumulative projected GB at each horizon, and the date each scenario crosses the contracted limit.

Methodology. Daily ingestion pulled from wandb-production.analytics.fct_weave_project_storage filtered by account_id, excluding feedback and call_exception storage types. Each scenario is a least-squares linear fit on a different lookback window, projected daily for 365 days. No seasonality or saturation modeling — these are deliberately simple linear trajectories. Treat them as scenarios for conversation, not point predictions.

Caveats. Forecasts are linear extrapolations of a non-stationary signal; real ingestion patterns are spiky. The contracted limit is read from dim_opportunities.total_weave_data_ingestion_limit for the current contract period — confirm against the live contract before any commercial conversation. Project-level breakdown is unavailable for dedicated-cloud Weave (project IDs are local to the customer's clickhouse instance and do not resolve via dim_projects).
