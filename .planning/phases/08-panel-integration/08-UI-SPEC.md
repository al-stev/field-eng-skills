# Phase 8 -- UI Design Contract

> Visual and interaction contract for integrating 9 deep-analytics page types as dashboard panels. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (self-contained HTML, no build step) |
| Preset | not applicable |
| Component library | none (vanilla JS + ECharts v5) |
| Icon library | Inline SVG (feather-style, stroke-based, 24x24 viewBox) |
| Fonts | Instrument Serif (display), Outfit (body), JetBrains Mono (labels/code) |
| CDN | `https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js` with local fallback |
| Theme | `echarts.registerTheme('wandb', {...})` via chart-helpers.js |

> Source: Detected from shell.html, chart-helpers.js, base-template.html. No React/shadcn/Tailwind in this project.

| Role | Font Family | Size | Weight | Line Height | Letter Spacing | Usage |
|------|-------------|------|--------|-------------|----------------|-------|
| Display | Instrument Serif | 24px | 400 | 1.2 | -0.3px | Panel titles (`.placeholder-title`), dashboard header h1 |

Additional type rules:

- ECharts tooltip body: Outfit 13px (set in wandb theme)
- No other font sizes permitted in new panels

> Source: Extracted from shell.html, support.js PANEL_CSS, chart-helpers.js wandb theme registration. The system uses exactly 3 fonts, 4 size stops (11, 14, 24, 28), and 2 weights (400, 600).

### Accent (10%)

| Token | Dark Value | Light Value |
|-------|------------|-------------|
| `--accent` | #d4a853 | #b8922e |
| `--accent-dim` | rgba(212,168,83,0.12) | rgba(184,146,46,0.10) |
| `--accent-border` | rgba(212,168,83,0.25) | rgba(184,146,46,0.25) |

**Accent reserved for:**
- Active sidebar nav item (border-left + text color + background)
- Active nav badge highlight
- Gold rule decorative element
- Toast notification border + text
- Breadcrumb link color
- First color in ECharts theme palette
- Overview panel "highlight" items

**Accent is NOT used for:** general buttons, links, form focus rings, or chart data series (those use semantic colors below).

### Semantic colors

Each semantic color has three tokens: base, dim (background fill), border (subtle outline).

| Semantic | Token | Dark Base | Reserved For |
|----------|-------|-----------|--------------|
| Positive | `--green` / `--green-dim` / `--green-border` | #4ade80 | Good health, resolved, high retention, fresh SDK, positive trend |
| Warning | `--amber` / `--amber-dim` / `--amber-border` | #fbbf24 | Medium risk, stale, declining, needs attention |
| Danger | `--red` / `--red-dim` / `--red-border` | #f87171 | High risk, churned, ancient SDK, error, critical attention items |
| Info | `--blue` / `--blue-dim` / `--blue-border` | #60a5fa | Links (Jira URLs), informational badges, secondary chart series |
| Neutral | `--orange` / `--orange-dim` / `--orange-border` | #fb923c | Tertiary chart series, engagement decay midpoints |
| Muted | `--gray` / `--gray-dim` / `--gray-border` | #6b7280 | Disabled states, no-data placeholders, inactive chart elements |

## Component Inventory (Panel Anatomy)

Each new analytics panel follows this exact structure. Deviations are not permitted.

### Panel IIFE wrapper

```javascript
(function() {
  'use strict';
  var PANEL_CSS = '...';
  // helper functions
  PanelRegistry.register({
    id: '{panel-id}',
    render: function(container, data, config) { ... },
    getHeadlineStats: function(data) { return []; },
    getAttentionItems: function(data) { return []; }
  });
})();
```

### Required CSS classes (reuse across all 9 panels)

| Class | Usage | CSS Pattern |
|-------|-------|-------------|
| `.stats-strip` | KPI headline row | `display: grid; grid-template-columns: repeat(N, 1fr); gap: 16px; margin-bottom: 32px;` |
| `.stat-card` | Individual KPI card | `background: var(--bg-elevated); border: 1px solid var(--border-subtle); border-radius: 6px; padding: 20px;` |
| `.stat-value` | KPI hero number | Outfit 28px weight 600, line-height 1.1 |
| `.stat-label` | KPI label below value | JetBrains Mono 11px weight 600, uppercase, letter-spacing 1.5px, color `--text-tertiary` |
| `.stat-sub` | KPI subtitle | JetBrains Mono 11px weight 400, color `--text-tertiary` |
| `.section-label` | Chart section heading | JetBrains Mono 11px weight 600, uppercase, letter-spacing 1.5px, margin-bottom 16px |
| `.time-period` | Time range indicator | JetBrains Mono 11px weight 400, color `--text-tertiary`, margin-top -8px, margin-bottom 12px |
| `.two-col` | Two-column chart layout | `display: grid; grid-template-columns: 1fr 1fr; gap: 24px; margin-bottom: 32px;` |
| `.panel-card` | Chart container card | `background: var(--bg-elevated); border: 1px solid var(--border-subtle); border-radius: 6px; padding: 24px;` |
| `.full-width` | Full-width section | `margin-bottom: 32px;` |
| `.sql-copy-btn` | Data provenance button | 20x20px, opacity 0.4 default, 1.0 on hover, inline SVG database icon |

### Chart container pattern

```html
<div id="{panelId}-{chartName}" style="width:100%;height:{N}px;"></div>
```

Heights per chart type:
- Bar/line/area charts: 320px
- Heatmaps: 400px (may vary with data rows)
- Treemaps: 360px
- Sankey diagrams: 400px
- Gauge charts: 200px
- Radar charts: 300px
- Scatter plots: 360px
- Funnel charts: 300px
- Sparkline grids: 40px per row

### Chart creation pattern

```javascript
var chart = ChartHelpers.createChart(document.getElementById('{id}'));
chart.setOption({ /* ECharts config */ });
```

All charts MUST use `ChartHelpers.createChart()` -- never raw `echarts.init()`.

### getHeadlineStats return shape

```javascript
[
  { label: 'Metric Name', value: '42%', color: 'var(--green)' },
  { label: 'Another', value: '7', color: 'var(--amber)' }
]
```

- Maximum 4 stats per panel
- `color` must reference a CSS custom property
- `value` is always a formatted string (not raw number)

### getAttentionItems return shape

```javascript
[
  {
    severity: 'high',     // 'high' | 'medium' | 'low'
    text: 'Human-readable concern',
    action: { panel: '{panel-id}' }
  }
]
```

- `severity: 'high'` items count toward tab title badge
- `action.panel` links to the relevant panel from the Overview

### isDark() helper

Each panel must include this dark mode detection helper:

```javascript
function isDark() {
  return window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches;
}
```

> Source: support.js, usage.js, panel-registry.js, shell.html sidebar/content rendering logic.

## Empty State Contract

Per D-09 and D-10 from CONTEXT.md, panels ALWAYS appear in the sidebar. When data is unavailable, the panel body shows a styled empty state.

### Full empty state (no data at all)

```html
<div class="placeholder-panel">
  <div class="placeholder-icon">
    <svg><!-- panel-specific icon --></svg>
  </div>
  <div class="placeholder-title">{Panel Name}</div>
  <div class="placeholder-desc">{Explanation of WHY data is unavailable}</div>
</div>
```

Styling (already in shell.html):
- Centered flex column, min-height 400px
- Icon: 48x48px rounded square, `--bg-surface` background, `--border-subtle` border
- Title: Instrument Serif 24px
- Description: Outfit 14px, `--text-tertiary`, max-width 400px, line-height 1.6

### Per-panel empty state copy

| Panel | Empty State Heading | Empty State Body |
|-------|-------|------|
| User Journey | User Journey | No user adoption data available. Ensure the customer has an sfdc_account_id configured in customers.yaml and dim_users data exists in BigQuery. |
| Cohort Analysis | Cohort Analysis | No retention cohort data available. This requires at least 2 months of user activity data in BigQuery. |
| Engagement Decay | Engagement Decay | No engagement decay data available. This requires daily engagement score history in BigQuery. |
| Team Detection | Team Detection | Team data not available. For dedicated cloud deployments, telemetry is anonymized and does not include team or entity fields. |
| Feature Velocity | Feature Velocity | No product area velocity data available. Ensure the customer has usage event data in BigQuery. |
| SDK Versions | SDK Versions | No SDK version data available. This requires cli_version or local_version fields in usage events. |
| Usage Correlation | Usage Correlation | No cross-account correlation data available. This analysis requires data from 10+ accounts to ensure privacy. |
| Risk Scoring | Risk Scoring | No risk scoring data available. This requires engagement score history and optionally renewal_predictions data. |
| Performance | Performance | No performance data available. Application performance metrics may not be available for all deployment types. |

### Partial data state

When some data subsections are available but others are missing, render available charts and show inline placeholders for missing sections:

```html
<div class="panel-card" style="text-align:center;padding:32px;">
  <div style="font-family:var(--font-mono);font-size:11px;font-weight:600;text-transform:uppercase;letter-spacing:1.5px;color:var(--text-tertiary);">
    {Section Name}
  </div>
  <div style="font-size:14px;color:var(--text-tertiary);margin-top:8px;">
    Data not available for this section
  </div>
</div>
```

## Interaction Patterns

### Panel-on-demand rendering

Panels render only when first navigated to (existing pattern in shell.html `navigateTo()`). This is critical for performance with 15 panels -- the shell must NOT render all panels on load.

### Chart resize

All ECharts instances are tracked by `ChartHelpers._instances[]` and resized via `ChartHelpers.resizeAll()` on:
- Window resize event
- Panel navigation (50ms delay after panel becomes visible)

### Data zoom (time series)

Analytics panels with 12+ months of data must include ECharts dataZoom:

```javascript
dataZoom: [
  { type: 'inside' },
  { type: 'slider', height: 20, bottom: 0 }
]
```

Applies to: Cohort trend, Engagement trend, Feature velocity trend, SDK version trend, Risk trend, Performance trend.

### Tooltip pattern

- Time series: `tooltip: { trigger: 'axis' }` with custom formatter
- Heatmaps/scatter/treemap: `tooltip: { trigger: 'item' }` with custom formatter
- Tooltip background uses `--chart-tooltip-bg` and `--chart-tooltip-border` tokens

### SQL copy buttons (data provenance)

Each chart section includes a SQL copy button (existing pattern from Phase 6 UX Polish). The button copies the BQ query that produced that data section to the clipboard, with a toast notification.

### Responsive breakpoints

| Breakpoint | Behavior |
|------------|----------|
| > 900px | Full sidebar (220px) with labels and badges |
| <= 900px | Collapsed sidebar (56px icon-only) |
| <= 700px | Stats-strip wraps to 2-col, two-col layout stacks to 1-col |

New panels must include the responsive media query overrides in their PANEL_CSS:

```css
@media (max-width: 700px) {
  .stats-strip { grid-template-columns: repeat(2, 1fr); }
  .two-col { grid-template-columns: 1fr; }
}
```

### Reduced motion

The shell already includes a `prefers-reduced-motion: reduce` media query that suppresses animations. New panels should not add custom animations beyond the existing `panelFadeIn` keyframe.
