# Phase 1 -- UI Design Contract

> Visual and interaction contract for the foundation HTML template. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase produces a single `base-template.html` that exercises every cross-cutting (XCUT) requirement with sample data. All 9 deep-analytics pages in Phases 2-4 inherit this contract. The template is self-contained HTML with inline CSS/JS and ECharts v5 via CDN -- no framework, no build step, no component library.

## Spacing Scale

Declared values (multiples of 4 only). Extracted from existing production templates.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, copy button padding-block |
| sm | 8px | Badge padding, compact element spacing |
| md | 16px | Default element spacing, chart container margins, nav link gaps, empty state margin-top |
| lg | 24px | Section padding, page-nav margin-bottom, KPI row gaps |
| xl | 32px | Empty state padding, major section breaks |
| 2xl | 48px | Page body padding-inline (via clamp), section group spacing |
| 3xl | 64px | Page-level vertical spacing between major regions (header, content, footer) |

Exceptions:
- Chart containers use `height: 400px` default (not a spacing token; varies per chart type in Phases 2-4).
- Page max-width is `1160px` (layout constraint, not spacing token). Source: design-system.md.
- Gold rule separator under title is `2px` height (decorative, not spacing).

## Typography

All fonts loaded via Google Fonts CDN. Weights are constrained to what the font import loads.

| Role | Font Family | Size | Weight | Line Height | Usage |
|------|------------|------|--------|-------------|-------|
| Display | Instrument Serif, Georgia, serif | clamp(28px, 4vw, 42px) | 400 | 1.2 | Page title (customer name), section headings |
| Body | Outfit, system-ui, sans-serif | 14px | 400 | 1.5 | Paragraph text, AI narrative content, empty state messages, tooltips (13px in ECharts theme) |
| Label / Heading | Outfit, system-ui, sans-serif | 16px | 600 | 1.3 | KPI labels, section sub-headings, nav link text (13px), legend text (12px in ECharts theme) |
| Micro-label | JetBrains Mono, Fira Code, monospace | 11px | 400 | 1.4 | Axis labels, chart labels, badge text, date stamps, "ANALYTICS" nav label, copy button text. Always uppercase with letter-spacing: 1.5px |

Loaded weights per font:
- Instrument Serif: 400 only
- Outfit: 300, 400, 500, 600, 700
- JetBrains Mono: 400, 500, 600

ECharts theme typography (registered via `echarts.registerTheme('wandb', ...)`):
- Default text: JetBrains Mono, 11px, `--text-secondary` color
- Title text: Outfit, `--text-primary` color
- Legend text: Outfit, 12px, `--text-secondary` color
- Tooltip text: Outfit, 13px, `--text-primary` color
- Axis labels: JetBrains Mono, 11px, `--text-tertiary` color

## Color

Dark mode is the default. Light mode activates via `@media (prefers-color-scheme: light)`. No manual toggle.

### Surface Hierarchy (60/30/10)

| Role | Dark Mode | Light Mode | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `#0c0f14` (`--bg-primary`) | `#f6f4f0` | Page background |
| Secondary (30%) | `#1a1f2b` (`--bg-surface`) / `#141820` (`--bg-elevated`) | `#eeece8` / `#ffffff` | Chart containers, nav bar background, empty state card, AI narrative container |
| Accent (10%) | `#d4a853` (`--accent`) | `#b8922e` | See reserved-for list below |

### Accent Reserved For (exhaustive list)

The gold accent (`--accent`) is used ONLY for these specific elements:
1. Active navigation link text color in the page nav bar
2. Gold rule separator (2px line under page title)
3. Copy button text on "Copied!" success feedback revert (uses `--green` during feedback, then back to default)
4. ECharts theme color palette position 2 (second series color: blue, **accent**, green, amber, red)
5. Inline stat value emphasis (KPI headline numbers that need highlight)

Accent is NEVER used for: all interactive elements, all buttons, all links, body text, backgrounds (except `--accent-dim` for active pill states in future phases).

### Semantic Colors

| Token | Dark Mode | Light Mode | Usage |
|-------|-----------|------------|-------|
| `--green` | `#4ade80` | `#1a7a4c` | Success states, "Copied!" button feedback, resolved/positive indicators |
| `--blue` | `#60a5fa` | `#1565a0` | Primary chart series color, active/in-progress indicators |
| `--amber` | `#fbbf24` | `#9a6e08` | Warning states, declining indicators, waiting states |
| `--red` | `#f87171` | `#dc2626` | Error states, cold/churned indicators, critical severity |

Each semantic color has `-dim` (rgba ~10% opacity) and `-border` (rgba ~22% opacity) variants for badge/pill backgrounds and borders.

### Text Colors

| Token | Dark Mode | Light Mode | Usage |
|-------|-----------|------------|-------|
| `--text-primary` | `#e8eaed` | `#1a1a1a` | Headings, body text, tooltip text |
| `--text-secondary` | `#8b92a0` | `#5c5c5c` | Meta text, chart default text, legend |
| `--text-tertiary` | `#5c6370` | `#8c8c8c` | Timestamps, axis labels, inactive nav links, empty state text |

### Border Colors

| Token | Dark Mode | Light Mode | Usage |
|-------|-----------|------------|-------|
| `--border` | `#2a3040` | `#d4d2ce` | Dividers, nav bar bottom border, empty state dashed border |
| `--border-subtle` | `#1e2430` | `#e0ded9` | ECharts axis lines, split lines, tooltip border |

### Other

| Token | Dark Mode | Light Mode | Usage |
|-------|-----------|------------|-------|
| `--bg-hover` | `#222838` | `#e6e4e0` | Hover states on interactive elements |
| `--radius` | `6px` | `6px` | Border radius for tooltips, empty state cards, buttons |
| `--noise-opacity` | `0.025` | `0` | SVG fractalNoise body::before overlay (dark mode only) |
| `--mode` | `dark` | `light` | JS-readable mode indicator |

### ECharts Color Palette (series order)

```javascript
// Dark: ['#60a5fa', '#d4a853', '#4ade80', '#fbbf24', '#f87171']

```

## Interaction Contract

### Must-Have Interactions (Phase 1)

| Interaction | Implementation | Requirement |
|-------------|---------------|-------------|
| ECharts tooltip on hover | `tooltip: { trigger: 'axis' }` for time-series, `trigger: 'item'` for individual points. Outfit font, 13px, 6px radius, elevated background with shadow. | XCUT-08 |
| Save chart as PNG | `toolbox: { feature: { saveAsImage: { title: 'Save as PNG', pixelRatio: 2, backgroundColor: bgElevated } } }` on every chart. Icon positioned top-right (right: 16, top: 8). | XCUT-09 |
| Copy AI narrative | `navigator.clipboard.writeText()` on button click. Button styled as JetBrains Mono 11px, border 1px solid `--border`, 4px 10px padding. "Copied!" feedback in `--green` for 2 seconds. | XCUT-10 |
| Linked page navigation | `<a>` links with relative paths (`./YYYY-MM-DD-page-name.html`). Active page styled with `--accent` color; inactive with `--text-tertiary`. | XCUT-11 |
| Responsive chart resize | `window.addEventListener('resize', () => chart.resize())` on every ECharts instance | Existing pattern from all 3 templates |
| Empty state rendering | `renderEmptyState(section, reason)` function producing `.empty-state` div with reason-specific copy | XCUT-07 |
| Dark/light mode | CSS `@media (prefers-color-scheme: light)` overrides `:root` custom properties. JS `isDarkMode()` + `getThemeColors()` for ECharts theme registration. | XCUT-06 |

## Anti-Patterns (Forbidden)

These patterns are explicitly banned. Source: `design-system.md` Anti-Patterns section.

| Anti-Pattern | What to Do Instead |
|-------------|-------------------|
| KPI cards (big number + small label in card grid) | Inline stat rows: value + label pairs, flex layout |
| Colored top border accents on cards | No cards; use sections with subtle borders |
| Everything-in-cards | Flatten the hierarchy; use sections and dividers |
| Sidebar navigation | Top page-nav bar with horizontal links |
| Zebra stripes on tables | Bottom border dividers only |
| Gradient text | Solid color text only |
| Inter / Roboto / Arial fonts | Instrument Serif + Outfit + JetBrains Mono only |
| Purple-to-blue gradients | W&B gold accent palette |
| Neon/glowing accents in dark mode | Tinted neutrals with subtle opacity |
| Bounce/elastic easing | `cubicOut` easing only |
| Sparklines as decoration | Sparklines only when showing real data (Phases 2-4) |
| Glassmorphism / frosted glass | Solid surfaces with noise texture |
| Chart.js or external charting library (non-ECharts) | ECharts v5 is the only permitted charting library |
| Emoji section headers | Text-only headers |
| Dark mode toggle button | CSS `prefers-color-scheme` only; no manual toggle |
| Templating engine (Jinja, Mako) | JS constant injection (`PAGE_DATA`, `AI_NARRATIVE`) |
| Shared CSS/JS include files | Each template is fully self-contained; duplicate tokens per file |

## Data Contract (Template <-> Python)

The HTML template receives two JavaScript constants injected at build time.

### PAGE_DATA Structure

```javascript
/* PAGE_DATA_START */
const PAGE_DATA = {
    customer: "Sample Corp",           // string: customer display name
    generated: "2026-03-24",           // string: ISO date of generation
    period: {
        start: "2025-03-24",           // string: analysis period start
        end: "2026-03-24"              // string: analysis period end
    },
    available: true,                    // boolean: false triggers empty state
    reason: null,                       // string|null: "no_data"|"config_error"|"api_error"|"schema_error"
    page_type: "user-journey",          // string: page identifier for nav highlighting
    kpis: [                             // array: 2-4 KPI objects for headline row
        { value: "--", label: "Total Users" },
        { value: "--", label: "Active (30d)" },
        { value: "--", label: "Top Product Area" },
        { value: "--", label: "Data Period" }
    ]
    // Page-specific data added in Phases 2-4
};
/* PAGE_DATA_END */
```

### AI_NARRATIVE Structure

```javascript
/* AI_NARRATIVE_START */
const AI_NARRATIVE = {
    executive_summary: "...",           // string: 2-3 sentence overview
    highlights: ["...", "..."],         // array of strings: key findings
    recommendations: ["...", "..."]     // array of strings: SE action items
};
/* AI_NARRATIVE_END */
```

Sentinel comment pairs (`/* *_START */` / `/* *_END */`) are used for machine-readable replacement boundaries. This avoids the regex-breaking-on-nested-objects pitfall documented in 01-RESEARCH.md.
