# Phase 4 -- UI Design Contract

> Visual and interaction contract for the Usage Correlation and Performance Deep Dive pages. Generated by gsd-ui-researcher.
>
> Phase 4 adds the final two analytical pages to complete the 9-page deep analytics suite. No new CSS tokens, fonts, or CDN dependencies are introduced. All visual rules from Phase 1 UI-SPEC carry forward. This document specifies ONLY what is new or page-specific for Phase 4: the privacy badge component, new chart layouts (scatter matrix, histogram/boxplot), cross-account anonymization patterns, the performance go/no-go gate, and page-specific copywriting.

## Privacy Badge Component Specification

A new visual component introduced in Phase 4 for the Usage Correlation page. This is the most visually prominent new element -- it must be unmistakable and non-dismissible.

### Rendering Logic

```javascript
function renderPrivacyBadge() {
    var c = getThemeColors();
    var container = document.getElementById('chartSection');
    var badge = document.createElement('div');
    badge.style.cssText = 'background:rgba(248,113,113,0.10); border:1px solid rgba(248,113,113,0.22); border-radius:6px; padding:8px 16px; margin-bottom:16px; display:flex; align-items:center; gap:12px;';

    var label = document.createElement('span');
    label.style.cssText = 'font-family:JetBrains Mono,monospace; font-size:11px; font-weight:600; letter-spacing:1.5px; text-transform:uppercase; color:' + c.red + '; white-space:nowrap;';
    label.textContent = 'SE-INTERNAL ONLY';

    var desc = document.createElement('span');
    desc.style.cssText = 'font-family:Outfit,sans-serif; font-size:14px; color:' + c.textSecondary + ';';
    desc.textContent = 'Cross-account analysis uses anonymized, pre-aggregated data. Minimum 10-account cohort enforcement. Never share this page with customers.';

    badge.appendChild(label);
    badge.appendChild(desc);
    container.insertBefore(badge, container.firstChild);
}
```

### Privacy Enforcement Rules

1. The privacy badge is rendered as the FIRST element in the chart section, before any charts.
2. The badge is NOT dismissible -- no close button, no collapse.
3. The badge is VISIBLE in print output (not hidden by `@media print`).
4. The page title includes no customer-identifying account IDs (display name only, as with all pages).
5. Peer account scatter points use generic labels ("Peer account") never real names or IDs.
6. Cohort sizes below 10 are suppressed -- the transform enforces this, the UI shows `insufficient_cohort` empty state.
7. The AI narrative must not reference specific peer account names.

## Anti-Patterns (Phase 4 Additions)

Inherited from Phase 1. These additional anti-patterns apply to Phase 4:

| Anti-Pattern | What to Do Instead |
|-------------|-------------------|
| Showing individual account names in peer data | Always anonymize: "Peer account" label, no names/IDs |
| Correlation matrix as full NxN (redundant) | Upper triangle only (symmetric matrix); diagonal shows self-correlation as "--" |
| Dismissible privacy badge | Privacy badge is permanent, non-dismissible, no close button |
| Stacked bar for latency distribution | Use single-series bar chart (histogram pattern) for clean bin comparison |
| 3D scatter for ARR-usage | 2D scatter with size/color encoding; 3D adds no value and reduces readability |
| Showing raw latency numbers without bins | Always bin latencies into human-readable ranges (0-1s, 1-2s, etc.) |
| Color-coding performance gauge with only red/green | Three-zone gradient: green (good) / amber (fair) / red (poor) for nuanced reading |
| Showing performance data without deployment context | Always display deployment_type in page metadata for data provenance |

## Go/No-Go Gate: Performance Deep Dive

Phase 4 includes an explicit go/no-go gate for the Performance Deep Dive page (documented in STATE.md blockers).

### Gate Logic

```
IF schema_validator confirms fct_application_performance AND fct_onscreen_loader_latencies exist
  AND query returns > 0 rows for the target account
THEN: proceed with full Performance Deep Dive rendering
ELSE: generate page with available: false, reason: "performance_descoped"
  -> renders descoped state layout (see above)
  -> AI narrative acknowledges descoping and explains why
  -> page is still navigable from other pages (nav link works, shows descoped message)
```

### Descoped Page Behavior

1. Page loads normally with nav bar, header, and KPI row (all values "--").
2. AI narrative section renders with a brief explanation of the descoping.
3. Instead of charts, a single centered empty-state div shows the descoped message.
4. The page is still included in linked navigation (link works, page exists).
5. Print output shows the descoped message (not hidden).
