class BaseTransform(ABC):
    @abstractmethod
    def transform(self, **dataframes: pd.DataFrame) -> dict[str, Any]:
        """Must include 'available' (bool) and 'reason' (str|None) keys."""
    def empty_result(self, reason: str = "no_data") -> dict[str, Any]:
        return {"available": False, "reason": reason}
```

    **transform() method logic:**

    1. **Early exits:** If cross_account is empty, return `self.empty_result("cross_account_unavailable")`. If `cross_account["account_id"].nunique() < self.MIN_COHORT_SIZE`, return `self.empty_result("insufficient_cohort")`.

    2. **Co-occurrence matrix:** Use pandas pivot_table to create account x product_area presence matrix (boolean: did account use this area?). Then compute co-occurrence: for each pair of product areas (upper triangle only), count accounts with both. Store as list of `[row_idx, col_idx, co_occurrence_pct, retention_pct, cohort_size]`. Suppress entries where cohort_size < MIN_COHORT_SIZE.

    3. **Retention overlay:** Define "retained" as account having activity in the most recent 30 days of the data. For each product combo, compute: (accounts with combo AND recent activity) / (accounts with combo) * 100. This is a simplification -- not true contract retention, but computable from activity data.

    4. **Account positioning:** Match `current_account_areas` against the correlation data. For each pair of areas the account uses, look up the co-occurrence retention rate. For pairs the account does NOT have, compute potential retention boost.

    5. **Next-best-action:** For each product area NOT in `current_account_areas`, compute the average retention lift if added (based on co-occurrence with the areas the account DOES use). Sort descending by retention_lift_pct. Format reasoning strings per UI-SPEC copywriting.

    6. **Expansion signals:** If `account_health` dict contains entitlement data (contracted_seats, weave_ingestion_cap, etc.), compare current usage to limits. If entitlement data is missing, return empty list (graceful degradation per Pitfall 6).

    7. **Peer benchmarking:** Filter arr_data by the current account's cs_tier. Compute product breadth percentile using `pandas.Series.rank(pct=True)`. Build distribution histogram (1 area, 2 areas, 3 areas, 4 areas, 5+ areas).

    8. **ARR-usage scatter:** From arr_data filtered to same tier, extract {breadth, arr} for peers (exclude current account). Current account gets its own entry. CRITICAL: NO account_id or account_name in any peer entry.

    9. **Privacy enforcement:** Before returning, verify no account_id values leaked. Use `import re; import json; serialized = json.dumps(result); assert not re.search(r'0018[A-Za-z0-9]{14}', serialized)`. If assertion fails, raise ValueError.

    10. **Narrative:** `_build_narrative()` returns dict with `executive_summary` (2-3 sentences on correlation insights), `highlights` (list of key findings), `recommendations` (list of actionable items).

    11. **KPIs:** Build kpis list: Product Areas Active (len of current_account_areas), Top Combo Retention (highest retention_pct in matrix), Breadth Percentile (from peer benchmarking), Expansion Signals (len of expansion_signals).

    **Return dict matching the PAGE_DATA shape in the interfaces section above.** Always include `page_type: "usage-correlation"`, `data_source: "pre-aggregated cross-account (min 10-account cohort)"`, `deployment_type` from kwargs.

    def _usage_correlation_handler(client, account_id, customer_name):
        """Usage Correlation -- cross-account product co-occurrence with privacy controls."""
        from queries import (cross_account_product_areas_query,
                             cross_account_arr_breadth_query,
                             product_areas_query, account_health_query)
        from bq_client import run_query
        from transforms.usage_correlation import UsageCorrelationTransform

        # Cross-account queries: NO account_id filter, higher bytes limit (100GB)
        cross_account_df = run_query(
            client, cross_account_product_areas_query(),
            maximum_bytes_billed=100_000_000_000
        )

        arr_data_df = run_query(
            client, cross_account_arr_breadth_query(),
            maximum_bytes_billed=100_000_000_000
        )

        # Single-account query: current account's product areas
        current_areas_df = run_query(
            client, product_areas_query(),
            account_id=account_id, maximum_bytes_billed=50_000_000_000
        )
        current_account_areas = (
            current_areas_df["product_area"].unique().tolist()
            if not current_areas_df.empty else []
        )

        # Account health for expansion signals + deployment type
        health_df = run_query(
            client, account_health_query(),
            account_id=account_id, maximum_bytes_billed=50_000_000_000
        )
        account_health = {}
        deployment_type = "Unknown"
        if not health_df.empty:
            row = health_df.iloc[0]
            account_health = row.to_dict()
            deployment_type = str(row.get("deployment_type", "Unknown"))

        transform = UsageCorrelationTransform()
        return transform.transform(
            cross_account=cross_account_df,
            arr_data=arr_data_df,
            current_account_areas=current_account_areas,
            account_health=account_health,
            customer_name=customer_name,
            account_id=account_id,
            deployment_type=deployment_type,
        )
    ```
