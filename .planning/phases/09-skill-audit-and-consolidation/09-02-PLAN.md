    Per D-06 and D-07, standardize all 35 SKILL.md files to have consistent sections.

    **Standard format** (per D-07). Every SKILL.md must have these sections in this order. The YAML frontmatter (name, description, argument-hint, allowed-tools, etc.) stays unchanged -- only the markdown body below the frontmatter is standardized:

    1. **Title** (# Skill Name) -- already present in all, keep as-is
    2. **Purpose** -- 1-2 sentence description of what the skill does and who uses it. Many files already have this as the first paragraph; keep or tighten.
    3. **Prerequisites** -- Credentials and setup needed. If skill has a setup-skill, reference it.
    4. **Usage** -- CLI commands or invocation patterns. Include concrete examples.
    5. **Parameters** -- Table of flags/arguments with descriptions. Skip if skill has no CLI parameters.
    6. **Output** -- What the skill produces (file paths, terminal output, etc.)
    7. **Dependencies** -- Other skills this consumes or is consumed by.
    8. **Credentials** -- Table of credential variables needed (from ~/.tsm-ai/.env).

    **Rules for the pass:**

    - Keep YAML frontmatter (---) blocks unchanged.
    - Keep any skill-specific sections that add value (e.g., "Anti-Patterns", "Design Rules", "Dedicated Cloud Customers") AFTER the standard sections.
