# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Cohort Analysis, Team Detection, and Risk Scoring pages. Generated by gsd-ui-researcher.
>
> Phase 3 adds three analytical pages to the existing `base-template.html` via the established `PAGE_RENDERERS` dispatch pattern. No new CSS, fonts, or CDN dependencies are introduced. All visual rules from Phase 1 UI-SPEC carry forward. This document specifies ONLY what is new or page-specific for Phase 3: new chart types (heatmap, gauge, radar, treemap), new data tables, new empty/staleness states, and page-specific copywriting.

## Spacing Scale

Inherited from Phase 1 UI-SPEC without modification.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding-block |
| sm | 8px | Badge padding, compact element spacing, table cell vertical padding |
| md | 16px | Default element spacing, chart container margins |
| lg | 24px | Section padding, narrative container padding, inter-chart gaps |
| xl | 32px | Empty state padding, staleness banner padding, major section breaks |
| 2xl | 48px | Page body padding-inline (via clamp) |
| 3xl | 64px | Page-level vertical spacing between major regions |

Exceptions:
- Heatmap chart container height: 480px (needs more vertical space for 12+ cohort rows).
- Gauge chart container: 300px height x 300px width (square aspect ratio for dial readability).
- Radar chart container: 360px height x 360px width (square for circle geometry).
- Team activity bar chart: 400px height (standard).
- Team x product area heatmap: 320px height (fewer rows than cohort heatmap).
- Staleness banner: 12px vertical padding, 16px horizontal padding (compact warning bar).

## Interaction Contract

### Inherited Interactions (from Phase 1)

All Phase 1 interactions carry forward: tooltips, saveAsImage toolbox, copy-to-clipboard, linked navigation, responsive resize, empty state rendering, dark/light mode.

### New Interactions for Phase 3

| Interaction | Implementation | Requirement |
|-------------|---------------|-------------|
| Heatmap cell hover | `tooltip: { position: 'top', formatter: function }` showing cohort label, period, retention %, and user count | CHRT-01, XCUT-08 |
| Heatmap visualMap interactive | `visualMap: { type: 'continuous', calculable: true }` -- user can drag range handles to filter visible retention ranges | CHRT-01 |
| Gauge pointer animation | `series.detail.valueAnimation: true` -- score number counts up on page load | RISK-01 |
| Radar hover highlight | `tooltip: { trigger: 'item' }` -- hovering radar point shows factor name and value | RISK-06 |
| Staleness banner rendering | Conditional DOM insertion before chart section when `inference_timestamp` is older than 30 days; uses amber warning styling | RISK success criteria |
| No-churn-model banner | Conditional DOM insertion when `churn_model_available === false`; uses blue informational styling | Risk fallback requirement |
| Data table scrolling | `max-height: 400px; overflow-y: auto;` on team breakdown and factor breakdown table containers | TEAM-01, RISK-02 |
| Data table sort | None -- tables are pre-sorted by the transform (events descending for teams, weight descending for risk factors). No client-side sort. | Simplicity; pre-sorted data |
| ECharts DataZoom on lifecycle | `dataZoom: [{ type: 'inside' }, { type: 'slider' }]` on user lifecycle stacked area (12+ months of data) | CHRT-05 |
| ECharts DataZoom on risk trend | `dataZoom: [{ type: 'inside' }, { type: 'slider' }]` on risk trend line (6 months) | RISK-03 |
| Team empty state branching | Three-tier empty state: (1) no data at all, (2) team names unavailable, (3) full team data available | TEAM-04 |

### Deferred Interactions

| Interaction | Deferred Because |
|-------------|-----------------|
| Click cohort row to filter lifecycle chart | Cross-chart filtering adds complexity; not in requirements |
| Drill-down from team to individual users | Would require expandable rows; not in requirements |
| Risk score simulation ("what-if") | Would require client-side recomputation; violates read-only |

## Staleness Banner Component Specification

A new visual component introduced in Phase 3 for the Risk Scoring page. This is NOT a CSS class (self-contained HTML constraint) but an inline-styled DOM element pattern.

### Rendering Logic

```javascript
function renderStalenessBanner(type, data) {
    // type: "stale_model" | "no_model" | "veto"
    var c = getThemeColors();
    var configs = {
        stale_model: {
            bg: 'rgba(251,191,36,0.10)',    // --amber-dim
            border: 'rgba(251,191,36,0.22)', // --amber-border
            color: c.amber,
            text: 'Churn model data is ' + data.age_days + ' days old. Interpret ML-based risk factors with caution.'
        },
        no_model: {
            bg: 'rgba(96,165,250,0.10)',     // --blue-dim
            border: 'rgba(96,165,250,0.22)', // --blue-border
            color: c.blue,
            text: 'Risk score excludes ML churn model (data unavailable). Score is based on behavioral signals only: engagement trend, seat utilization, and support velocity.'
        },
        veto: {
            bg: 'rgba(248,113,113,0.10)',    // --red-dim
            border: 'rgba(248,113,113,0.22)', // --red-border
            color: c.red,
            text: 'Critical threshold exceeded: ' + data.factor_name + ' triggered automatic high-risk floor.'
        }
    };
    var cfg = configs[type];
    // Banner: 12px padding-block, 16px padding-inline, radius 6px, 13px Outfit weight 500
    return '<div style="background:' + cfg.bg + '; border:1px solid ' + cfg.border +
           '; border-radius:6px; padding:12px 16px; margin-bottom:16px; font-family:Outfit,sans-serif; font-size:13px; font-weight:500; color:' + cfg.color + ';">' +
           cfg.text + '</div>';
}
```

Phase 3 introduces data tables for Team Detection and Risk Scoring. These follow the pattern established by the SDK Versions page in Phase 2.

### Table Styling Rules

| Property | Value | Source |
|----------|-------|--------|
| Table width | 100% | Full container width |
| Border collapse | collapse | No double borders |
| Font | Outfit, 13px body; JetBrains Mono, 11px headers | Phase 1 typography |
| Header font-weight | 400 | NOT bold; headers are subtle micro-labels |
| Header text-transform | uppercase | Micro-label pattern |
| Header color | `--text-tertiary` | Subtle header labels |
| Cell padding | 8px 12px | sm vertical, sm+xs horizontal |
| Row border | bottom only, 1px solid `--border-subtle` | No zebra stripes (anti-pattern) |
| Body text color | `--text-primary` for primary column, `--text-secondary` for secondary data | Hierarchy emphasis |

| Badge cells | `padding: 2px 8px; border-radius: 10px; font-size: 11px; font-family: JetBrains Mono` | Reuses SDK Versions badge pattern |

## Anti-Patterns (Phase 3 Additions)

Inherited from Phase 1. These additional anti-patterns apply to Phase 3:

| Anti-Pattern | What to Do Instead |
|-------------|-------------------|
| Treemap for team hierarchy | Bar charts + heatmap; treemap requires size data that team detection does not naturally produce |
| Risk gauge with 180-degree arc | Use 220-degree arc (startAngle: 200, endAngle: -20) for better visual balance |
| Equal weighting in risk composite | Use asymmetric weights: churn 40%, engagement 25%, utilization 20%, support 15% |
| Showing raw churn probability to SE | Always show normalized 0-100 risk score; raw probability in factor breakdown only |
| Red/green only heatmap gradient | Three-stop gradient: red-amber-green for nuanced reading |
| Stacked bar for lifecycle | Stacked area with areaStyle for smoother time-series visualization |
