    Add 6 new query functions to `.claude/skills/bigquery/scripts/queries.py`, each following the existing `_ref()` + parameterized SQL pattern:

    1. **`cohort_retention_query()`** -- Compute cohorts from `ext_daily_user_event_usage`. This is the fallback Strategy B from research (always available, unlike `agg_weekly_user_retention_features`). SQL structure:
       - CTE `user_first_activity`: SELECT universal_user_id, FORMAT_DATE('%Y-%m', MIN(date_day)) AS cohort_month FROM _ref("ext_daily_user_event_usage") WHERE account_id = @account_id AND date_day >= DATE_SUB(CURRENT_DATE(), INTERVAL 18 MONTH) AND event_count > 0 GROUP BY universal_user_id
       - CTE `user_monthly_activity`: SELECT DISTINCT u.universal_user_id, u.cohort_month, FORMAT_DATE('%Y-%m', e.date_day) AS active_month FROM user_first_activity u JOIN _ref("ext_daily_user_event_usage") e ON u.universal_user_id = e.universal_user_id WHERE e.account_id = @account_id AND e.event_count > 0
       - Final SELECT: cohort_month, active_month, COUNT(DISTINCT universal_user_id) AS active_users GROUP BY cohort_month, active_month ORDER BY cohort_month, active_month

    2. **`user_lifecycle_query()`** -- Monthly New/Retained/Resurrected/Churned from `agg_daily_user_activity`. SQL structure:
       - SELECT FORMAT_DATE('%Y-%m', date_day) AS month, SUM(CASE WHEN user_has_any_event_accounting = 'new' THEN 1 ELSE 0 END) AS new_users, SUM(CASE WHEN ... = 'retained' THEN 1 ELSE 0 END) AS retained, SUM(CASE WHEN ... = 'resurrected' THEN 1 ELSE 0 END) AS resurrected, SUM(CASE WHEN ... = 'churned' THEN 1 ELSE 0 END) AS churned
       - FROM _ref("agg_daily_user_activity") WHERE account_id = @account_id AND date_day >= DATE_SUB(CURRENT_DATE(), INTERVAL 18 MONTH)
       - GROUP BY month ORDER BY month

    3. **`team_detection_query()`** -- Team breakdown from org_name. SQL:
       - SELECT COALESCE(org_name, 'Unknown') AS team_name, COUNT(DISTINCT universal_user_id) AS member_count, SUM(event_count) AS total_events, MIN(date_day) AS first_active, MAX(date_day) AS last_active, COUNT(DISTINCT CASE WHEN org_name IS NOT NULL THEN org_name END) OVER() AS distinct_teams, MAX(CASE WHEN is_part_of_team THEN 1 ELSE 0 END) OVER() AS has_team_flag
       - FROM _ref("ext_daily_user_event_usage") WHERE account_id = @account_id AND date_day >= DATE_SUB(CURRENT_DATE(), INTERVAL 12 MONTH) AND event_count > 0
       - GROUP BY team_name ORDER BY total_events DESC

    4. **`team_champions_query()`** -- Most active user per team with identity resolution. SQL:
       - Uses identity_resolution_cte() pattern
       - SELECT org_name AS team_name, universal_user_id, username, email, SUM(event_count) AS total_events, MAX(date_day) AS last_active
       - FROM _ref("ext_daily_user_event_usage") WHERE account_id = @account_id AND date_day >= DATE_SUB(CURRENT_DATE(), INTERVAL 12 MONTH) AND org_name IS NOT NULL
       - QUALIFY ROW_NUMBER() OVER (PARTITION BY org_name ORDER BY SUM(event_count) DESC) = 1

    5. **`engagement_trend_query()`** -- Customer engagement score over 6 months for risk scoring. SQL:
       - SELECT FORMAT_DATE('%Y-%m', date_day) AS month, AVG(customer_engagement_score) AS avg_engagement_score, COUNT(DISTINCT universal_user_id) AS active_users
       - FROM _ref("agg_daily_customer_engagement_score") WHERE account_id = @account_id AND date_day >= DATE_SUB(CURRENT_DATE(), INTERVAL 6 MONTH)
       - GROUP BY month ORDER BY month

    6. **`risk_support_tickets_query()`** -- Support ticket count for risk factor. SQL:
       - SELECT COUNT(*) AS ticket_count_90d
       - FROM _ref("dim_helpdesk_tickets") WHERE account_id = @account_id AND ticket_created_date_day >= DATE_SUB(CURRENT_DATE(), INTERVAL 90 DAY)

    All functions return SQL strings with @account_id parameter. Use _ref() for all table references. Column-prune every query (no SELECT *).
