**Helper functions:**

- `classifyIssue(issue)` — Port from v1. Uses comments data to classify: if resolved status -> 'resolved'. If open/backlog/triage with no eng comments -> 'needs_triage'. If non-resolved with eng comment within STALE_DAYS -> 'active'. Otherwise -> 'stale'. Use `INTELLIGENCE_DATA.generated` as reference date (access global).

- `getStatusCategory(status)` — Return 'resolved', 'active', 'waiting', or 'triage' based on STATUS_CATEGORIES lookup.

- `getFiltered()` — Filter `_issues` by current `_filterState`: status pill filters by category, type filters by "Bug"/"Feature Request", theme filters by issue.theme, search filters by key/summary text match (case-insensitive), bucket applies health bucket filter. Also by default hide resolved issues unless status filter is 'resolved' or 'all'.

- `statusBadge(status)` — Return HTML span with colored pill based on status category.

- `priorityBadge(priority)` — Return HTML span: P0/P1 red, P2 amber, P3 gray.

- `ageBadge(dateStr)` — Return HTML span with days since date, colored green/amber/red.

- `daysBetween(dateStr)` — Compute days from dateStr to `INTELLIGENCE_DATA.generated`.

**Filter system:**

`buildFilters(container)` — Port v1 lines 1873-1970. Create filter bar with:
- Status pills: All, Active, Waiting, Resolved, Triage. Click sets `_filterState.status`, re-renders.
- Type pills: All, Bugs, Features. Click sets `_filterState.type`, re-renders.

```javascript
if (!data || !Array.isArray(data) || data.length === 0) return [];

  { label: 'Bugs', value: String(bugs.length), color: bugs.length > 0 ? 'var(--red)' : 'var(--text-tertiary)' },

];
```
