# Phase 05 — UI Design Contract

> Visual and interaction contract for Dashboard V2 — Modular Folder-Based Architecture. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS custom properties) |
| Preset | not applicable |
| Component library | none (vanilla JS DOM construction) |
| Icon library | inline SVG (18x18px default, 24x24px placeholder) |
| Font | Instrument Serif (display) + Outfit (body) + JetBrains Mono (mono/labels) |

**Source:** Established across v1 monolith (`intelligence-dashboard.html`), deep-analytics `base-template.html`, v2 prototype (`dashboard-v2.html`), and 6 support ticket prototypes. All templates share identical CSS custom property names and values.

**Font loading:** Google Fonts preconnect + stylesheet link. Identical across all existing templates.
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Instrument+Serif&family=JetBrains+Mono:wght@400;500;600&family=Outfit:wght@300;400;500;600;700&display=swap" rel="stylesheet">
```

**ECharts:** Bundled locally at `lib/echarts.min.js` (v5.x). No CDN dependency. Source: DASHBOARD-V2-SPEC.md architecture decision.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps (legend dots, badge internal padding vertical), gap between stacked bars |
| sm | 8px | Compact element spacing (nav separator margins, sidebar nav padding, filter group gaps) |
| md | 16px | Default element spacing (stat card gaps, section label margin-bottom, header padding, sidebar nav item padding) |
| lg | 24px | Section padding (panel card internal padding, two-col grid gap, panel content padding min) |
| xl | 32px | Layout gaps (header horizontal padding, stats strip margin-bottom, section breaks) |
| 2xl | 48px | Major section breaks (panel content padding max via clamp) |
| 3xl | 64px | Not used in dashboard shell (reserved for page-level spacing in deep-analytics standalone pages) |

Exceptions:
- Sidebar collapsed width: 56px (not on 8-point grid; dictated by 20px icon + 18px padding each side)
- Sidebar expanded width: 220px (content-driven, not grid-aligned)
- Header min-height: 64px (aligns to 3xl token)
- Nav item border-left: 3px accent indicator (visual weight, not spacing)
- Badge pill padding: 2px 7px (optical balance for 10px mono text)
- Scrollbar width: 6px (browser convention)

**Source:** Extracted from dashboard-v2.html prototype CSS (lines 112-500+). All values verified against actual usage.

### 60/30/10 Split (Dashboard Shell)

| Role | Proportion | Surfaces | Token |
|------|-----------|----------|-------|
| Dominant (60%) | Content area background | Panel area, body, empty states | `--bg-primary` |
| Secondary (30%) | Structural chrome | Header bar, sidebar, stat cards, panel cards, tooltips | `--bg-elevated` |
| Accent (10%) | Interactive highlights | See reserved-for list | `--accent` (gold) |

### Accent Reserved For (explicit list)

1. **Active sidebar nav item** — left border color + icon/text color + background tint
2. **Active filter pill** — background tint + border + text color
3. **Issue key links** — mono text colored with accent on hover
4. **Header rule** — gold 2px divider in header
5. **Internal mode toggle** — active state background tint + border + text
6. **Active filter indicator badge** — background tint + border + text
7. **Command palette footer shortcuts** — not accent, but keyboard shortcuts use surface bg
8. **Attention callout hover action text** — "View" action text on callout hover

Accent is NEVER used for: body text, chart data series (use semantic colors), table borders, non-active nav items, disabled states.

## Layout Contract

### Shell Grid

```
grid-template-rows: auto 1fr
grid-template-columns: var(--sidebar-width) 1fr   /* expanded: 220px 1fr */
grid-template-columns: var(--sidebar-collapsed) 1fr /* collapsed: 56px 1fr */
```

- Header spans full width (`grid-column: 1 / -1`), min-height 64px
- Sidebar is row 2 col 1, full height with `overflow-y: auto`
- Content area is row 2 col 2, `overflow-y: auto`, `overflow-x: hidden`
- Grid transition: `grid-template-columns 0.25s ease-out`
- Body: `overflow: hidden; height: 100vh` (no body scroll; content area scrolls)

### Panel Content Area

- Padding: `clamp(24px, 3vw, 40px) clamp(24px, 4vw, 48px)`
- Max width: `1100px` (content does not stretch to fill ultra-wide screens)
- Panel visibility: `display: none` default, `.active` sets `display: block`
- Panel entry animation: `panelFadeIn 0.3s ease-out` (opacity 0->1, translateY 8px->0)

### Responsive Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| > 900px | Full sidebar with expand/collapse toggle |
| <= 900px | Sidebar auto-collapses to 56px icon-only, labels/badges hidden |
| <= 700px | Stats strip becomes 2-col grid, two-col layout becomes single column, header wraps |

### Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

**Source:** dashboard-v2.html lines 112-777.

## Interaction Contract

### Sidebar Navigation

| Interaction | Behavior |
|-------------|----------|
| Click nav item | Hash changes to `#panel-id`, panel loads on demand (script tag inserted if not already), previous panel hidden, new panel shown with fade animation |
| Click toggle button | Shell toggles `.collapsed` class, grid animates from 220px to 56px, labels/badges fade to opacity 0 |
| Keyboard 1-6 | Jump to panel by order (Overview=1, Issues=2, Support=3, Usage=4, Actions=5, Slack=6) |
| Keyboard Cmd+K | Open command palette overlay |
| Panel with no data | Nav item not rendered (hidden from sidebar). Panel never loads. |
| Browser back/forward | Hash change triggers panel navigation (popstate listener) |
| Direct URL with hash | On page load, navigate to hash panel. Fallback to `#overview` if hash invalid. |

### Command Palette

| Interaction | Behavior |
|-------------|----------|
| Cmd+K / Ctrl+K | Opens overlay with search input focused |
| Type query | Filters panel list + data items (tickets, issues, actions) by fuzzy match |
| Arrow keys | Navigate results list (`.selected` class) |
| Enter | Navigate to selected result's panel (close palette) |
| Escape / click overlay | Close palette |

### Panel Loading

| Interaction | Behavior |
|-------------|----------|
| First visit to panel | `<script>` tag for panel JS inserted into DOM, panel calls `PanelRegistry.register()`, `render()` invoked with container + data |
| Subsequent visits | Panel already in DOM, just toggle `.active` class (no re-render) |
| Panel JS error | Console error logged, panel shows placeholder with "Panel failed to load" message |
| Window resize | `ChartHelpers.resizeAll()` fires, all tracked ECharts instances call `.resize()` |

### Audience Toggle

| Interaction | Behavior |
|-------------|----------|
| Click "Internal" | `.active-internal` class, accent-dim background, accent text. Internal data visible. |
| Click "External" | `.active-external` class, green-dim background, green text. Sensitive data hidden. |
| No toggle in v2 | Audience is set at generation time. `data.js` simply omits sensitive fields for external mode. Toggle is cosmetic/informational only. |

### Table Interactions

| Interaction | Behavior |
|-------------|----------|
| Row hover | Background changes to `--bg-hover` (0.15s transition) |
| Jira key click | Opens Jira issue in new tab (`https://wandb.atlassian.net/browse/WB-XXXXX`) |
| Subject cell overflow | `text-overflow: ellipsis` at max-width 300px, full text in tooltip |
| Theme section header click | Collapse/expand body (grid-template-rows 0fr/1fr animation, arrow rotates -90deg) |

### Filter Interactions (Issues Panel)

| Interaction | Behavior |
|-------------|----------|
| Click filter pill | Toggle `.active` class (accent styling), filter table rows |
| Select dropdown change | Filter by selected value |
| Search input | Live filter table rows by text match (debounced) |
| Active filter badge | Shows when filters applied, click to clear all |
| Search focus | Border changes to `--accent-border` |

## Panel Registry Contract

Every panel JS file MUST call `PanelRegistry.register()` with the following shape:

```javascript
PanelRegistry.register({
  id: 'string',           // matches panels.yaml id and URL hash
  group: 'string',        // 'intelligence' | 'usage' | 'activity'
  label: 'string',        // sidebar display name
  icon: 'string',         // SVG icon identifier
  badgeKey: 'string',     // dot-path into INTELLIGENCE_DATA for badge count (or null)
  dataKey: 'string',      // dot-path into INTELLIGENCE_DATA for data presence check (or null)

  render(container, data, config) {
    // container: DOM element (#panel-{id})
    // data: resolved data object from INTELLIGENCE_DATA
    // config: { audience: 'internal'|'external' }
    // Returns: { charts: [echarts_instances] } for resize handling
  },

  getHeadlineStats(data) {
    // Returns: [{ label: string, value: string, color: string }]
    // Used by Overview panel to aggregate stats strip
  },

  getAttentionItems(data) {
    // Returns: [{ severity: 'high'|'medium'|'low'|'info', text: string, action: { panel: string, filter?: string } }]
    // Used by Overview panel for attention callouts
  }
});
```

**Panel CSS scoping:** Each panel defines a `PANEL_CSS` string constant. The shell wraps all selectors with `#panel-{id}` prefix to prevent style collisions between panels.

## Noise Texture

Dark mode only (light mode sets `--noise-opacity: 0`):

```css
body::before {
  content: '';
  position: fixed;
  inset: 0;
  opacity: var(--noise-opacity); /* 0.025 dark, 0 light */
  background-image: url("data:image/svg+xml,...feTurbulence fractalNoise...");
  background-size: 256px;
  pointer-events: none;
  z-index: 9999;
}
```

Subtle film-grain effect. Must be present in `shell.html`. Applied via `::before` pseudo-element so it sits above all content but with `pointer-events: none`.
