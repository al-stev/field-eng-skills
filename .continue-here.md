2. **No new credentials** — Uses existing BQ ADC auth via `wandb-sa-sandbox`

4. **5 visualizations selected for dashboard**: headline stats strip, monthly volume trend, active tickets table, top concerns bar, top submitters bar
5. **Dropped from dashboard**: resolution donut (becomes a stat), escalation sankey (low info density)

## Remaining Work — Dashboard Refactor

### Architecture Decision: Option B — Panel Fragment Composition
Refactor the 3700-line monolith `intelligence-dashboard.html` into modular panel fragments:

```
customer-snapshot/templates/
  shell.html              -- Header, sidebar nav, content container, shared CSS tokens
  panels/
    overview.html         -- Aggregated headline stats from all panels
    issues.html           -- Jira issue list (extracted from current monolith)
    support-tickets.html  -- NEW: 5 support ticket visualizations
    usage.html            -- Seats/Weave/hours/radar (extracted from current monolith)
    actions.html          -- Asana SE actions (extracted from current monolith)
    slack.html            -- Sentiment/hot threads (extracted from current monolith)
  panels.yaml             -- Declarative panel registry
  compose.py              -- Assembles shell + panels → single self-contained HTML
```

### Sidebar Navigation (replaces long scroll)
- Collapsible sidebar with nav items per panel
- Badge counts on nav items (e.g., "Issues (5)", "Support (8)")
- Lazy rendering — panels build DOM on first navigation
- Deep linking via URL hash (#support, #usage)
- Missing data = hidden nav item (graceful degradation)
- Keyboard navigation (1-6 keys, j/k scroll, o to open links)

### Panel Contract
Each panel provides: `{ id, render(container, data), getHeadlineStats(), getAttentionItems() }`
- Overview page calls `getHeadlineStats()` on all panels
- Attention callouts call `getAttentionItems()` on all panels

### Structural Improvements
1. **panels.yaml manifest** — Declarative panel registry, adding a panel = file + YAML entry
2. **Scoped CSS via panel ID namespacing** — Auto-wrapped `#panel-xxx .selector { }`
3. **Cross-panel event bus** — 10-line pub/sub for linked data (Jira keys, Asana badges)
4. **Shared chart helpers** — Common ECharts tooltip/color/resize patterns extracted once
5. **Data transform layer** — Normalise BQ output before panels consume it
6. **Theme tokens as CSS custom properties** — Panels use `var(--color-warning)` not hardcoded hex
7. **Sample data fixtures per panel** — Each fragment can open standalone during development

### Delight Features
1. **Cmd+K command palette** — Search across all data, jump to any panel
2. **Sparkline badges in sidebar** — Mini trend per nav item
3. **"Since last dashboard" diffing** — Show changes since previous generation
4. **Print/export mode** — Cmd+P produces clean multi-page PDF
5. **"So what?" annotations** — AI-generated insight text below each chart
